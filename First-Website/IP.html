Internet Protocols

To clearly define a protocol is a vague term because of its ability to be so many things at the same time. But if we look back at the clear English term it can be explained by saying that a protocol refers to a code of conduct or a form of etiquette. In simpler terms it is rules of formality to be followed to ensure that communication is enforced equally without causing conflict. So, regarding networking, they define a language and a set of rules and procedures that gives devices and systems the ability to communicate with each other (Kozierok, 2005).
HTTP
1.HTTP (Hyper Text Transfer Protocol)

HTTP is a protocol that transfers web pages from a web server to a responding web client. Also defined as the set of rules required to exchange files or multimedia content over the Web (Tutmaster, 2019).

2.IP (Internet Protocol)

Users send and receive data from their local device, from here the data gets spliced into packets, one for the sender and one for the receiver. These are called packets; it goes to a gateway and gets directed until it reaches its target address (Goss, 2020).

3.TCP (Transmission control Protocol)

This protocol separates data into packets so that it can be shared over a network, commonly sent by technological devices such as switches and routers to get to the targets (CDW, 2019).

4.FTP (File Transfer Protocol)

This was designed to give access to programs and large data files. Mainly helps with the transfer of files from one set to another set (Tutmaster, 2019).
