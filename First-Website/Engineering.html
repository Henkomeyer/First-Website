Client-Side Engineering

The job of a client-side software engineer is to develop the code, applications and infrastructure that the client or user will be interacting with and where the retrieved results of the backend infrastructure is displayed for the viewing pleasure of the user.

As the name ‘client-side’ suggests, this is the software that the user installs onto their device. As such, it is the responsibility of the client-side web app engineer (also known as a frontend programmer) to create an accessible, user-friendly interface that meets the requirements of a wide variety of human wants and needs. Furthermore, because this software is being installed directly onto the client’s device, the software needs to protect the user’s personal data from malicious software and people. If this software were to have exploitable vulnerabilities, then it would prove dangerous to the client that has it installed on their machine.

Server-Side Engineering

Server-side software engineers is the other half of the dynamic that is a web application. They oversee the building, troubleshooting problems with, maintaining and repairing of the backend infrastructure, most notably the web server. These developers might need a broad repertoire of knowledge and experience because of the wide variety of elements that could be on the server-side of the application. A server-side software engineer needs to have skills in developing and administrating a database and a database management system, for example, or else they risk not being able to maintain a database at the backend of a web application.

Whereas client-side programmers are much more familiar with markup languages (like HTML, CSS and JavaScript), backend programmers are well versed in server programming languages (like Python, SQL and Java). Programmers that are educated in a wide variety of languages and are capable of both maintaining the front- and backend of a web application are called full-stack developers and are very sought after in the Information and Communication Technology industry.









































Defining Web App Architecture

Web Application Architecture is the scaffolding with which one can indicate how different web application entities relate to and interact with each other. Such entities include databases, middleware systems, graphic user interfaces and other applications. Having a predefined architecture allows applications to work at the same time and independently of each other.

The architecture can be divided into the frontend and the backend, where the frontend and the backend exchange information with one another to give the user what they want to be displayed. The frontend is mainly made up of the graphical user interface from where the user enters data. The backend encapsulates the applications that do what the user is requesting. This can involve entities like databases and file systems that all communicate with the web server. The web server, through the applications, then responds to the requests of the frontend with that which it requested. The user only observes the frontend, and cannot see the workings of the backend on the surface.

Component Description
Domain Name System (DNS) DNS is most comparable to a phone book for internet IP addresses, and is used similarly to locate and retrieve the address of the IP that the user requests
Load Balancer To avoid servers being overloaded by user requests, the load balancer reduces the load of a high-traffic server to a server that is not handling as many requests. This avoids unnecessary waiting times or unexpected problems that could result from an overloaded server.
Web Application Servers This component is located in the backend and communicates the request it receives from the frontend to the entities in the backend to search and find what it is supposed to retrieve.
Databases Databases allow web applications a way to store data in high volumes and in an orderly fashion that would make said data easily retrievable in the future.
Caching Service Requests made and retrieved are stored in cache memory to increase the speed at which that information is retrieved in the future.
[Optional] Job Queue A dual-part component, the job queue and job queue server monitor and schedule when certain tasks in the web application should be completed.
[Optional] Full-Text Search Service A technology that searches through the content of documents to return results that might be of interest to the user that made the request.
Services These are miscellaneous applications that large web applications (and their components/infrastructure) interact with
Data Warehouse Data warehousing is the procedure of storing copious amounts of data in a very compressed state, either for future reference or for study/analysis by database analytics.
Content Delivery System (CDN) CDN is tasked with the responsibility of displaying different file formats (like HTML, CSS, JavaScript, gifs and images) in the correct way that they were intended to be.
